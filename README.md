# Burn Your Own Style

ユーザーがWeb サイト制作で蓄積してきたスタイルシステム（クラス、変数、スタイリングの癖）を、Claude Code・Cursor 等のエージェントに理解させることで、Web 制作の全ての作業をエージェントに任せることを目標としたプロジェクトです。

完成後はフレームワークごとに作成されたブランチを丸ごとコピーして新規プロジェクトを作成することを想定しています。

UnitシステムとTailwind CSSの明確な役割分担により、エージェントが「構造」と「装飾」を分けて作業できるよう設計されています。

**現在は`react`ブランチで進行中。完成後にmainにマージ予定**

## Unitとは

Unitは、CSSクラス名・スキル名・コンポーネント名を兼ねるシステム固有の概念です。

## 思想

- LLM の学習データに基づく`webデザイン・コーディング`は平均的であり、振れ幅の大きい標準的ではない出力結果になるため、個人のマークアップ、スタイリングとかけ離れたものになる。
- 創造性を持たせずに事務作業のように決められた手順のみを実行させる環境を整えることでLLM単体では創造できないオリジナリティーのあるプロダクトを効率的に実装できる。
- 2026年3月時点のトレンド、主にLLMの自立性強化とは正反対のアプローチであるが、ウェブ制作のコモディティー化は業界全体の緩やかな品質低下につながると考え、ニッチでハイクオリティな需要を満たすための手段として環境構築に努める。

## 目的

- 個人のクラス、変数、スタイリングの癖を再利用可能にする
- エージェントによるデザイン再現の精度と効率を向上させる
- 一貫性のあるUI/UXを維持しながら開発スピードを上げる

## 特徴

| 特徴 | 説明 |
|-----|------|
| **Unitシステム** | CSSクラス名・スキル名・コンポーネント名を兼ねる単位で構造化 |
| **明確な役割分担** | Unit＝レイアウト骨格、Tailwind CSS＝装飾・色・微調整 |
| **少数のUnit** | 10〜20程度の少数Unitで運用を想定 |
| **マルチフレームワーク** | Astro、Next.js、Vue、SvelteKit、Viteなど主要フレームワークに対応予定 |

## 設計思想

速記性を重視し、Unitではパスカルケースを使うことでTailwindと視覚的に差別化

## クイックスタート

### 既存ブランチをクローンする

Burn Your Own Styleは、あらかじめ各フレームワークで構築されたテンプレートプロジェクトをクローンして使用します。

| フレームワーク | ブランチ | 説明 |
|------------|-------------|------|
| Astro | `astro` | Astro環境でのコンポーネント実装済み |
| Next.js | `next-js` | Next.js環境でのコンポーネント実装済み |
| SvelteKit | `sveltekit` | SvelteKit環境でのコンポーネント実装済み|
| Vue | `vue` | Vue+ vite環境でのコンポーネント実装済み |
| Vite | `vite` | Vite+React環境でのコンポーネント実装済み |
| react | `react` | Vite+React環境でのコンポーネント実装済み |

**使用手順**:

1. 目的のフレームワークのブランチをクローンまたはコピー
2. 依存パッケージをインストール（`npm install`、`pnpm install` など）
3. 開発サーバーを起動して確認
4. 新規ウェブサイト/ウェブアプリの開発を開始

## Unitシステム

| 呼び方 | 指しているもの |
|--------|--------------|
| `Cards クラス` | scss/globals.scss の `.Cards` クラス |
| `Cards スキル` | `.claude/skills/Cards/SKILL.md` |
| `Cards コンポーネント` | `Cards.tsx`など |
| `Cards ユニット` | 上記すべてを包含する総称 |

### Unit一覧

| Unit | 説明 |
|------|------|
| `Cards` | 一般的な横並びコンテナ（2〜5列程度） |
| `Panel` | 一般的な縦並びコンテナ。ImgTextを複数まとめたものに近い |
| `ImgText` | 画像+テキスト横並びコンテナ。画像比率をValueクラスで可変 |
| `Toggle` | summary,detailsタグを使う開閉コンテンツ。特殊なCSSが必要なためUnit化 |
| `FlexR` | クラス名が比率を表す。2種類のコンテンツの比率を制御するラッパー |

### Valueクラス

CSS変数に値を渡すクラス。

| クラス | 意味 | 使用Unit |
|--------|------|----------|
| `.col2`, `.col3`, `.col4` | カラム数 | Cards |
| `.img20`, `.img30`, `.img40`, `.img60` | 画像比率 | ImgText, Panel |

### Modifierクラス

Unitクラス毎に定義されている状態・モードの切り替え（パスカルケース）。

| クラス | 効果 | 使用Unit |
|--------|------|----------|
| `.IsRev` | 画像の位置を反転（右に） | ImgText, Panel |
| `.IsLayer` | 画像とテキストを重ねる | Cards |
| `.IsFlow` | パネル間に矢印 | Panel |
| `.IsQa` | Q&Aの装飾付与 | Toggle |
| `.IsGrow` | スマホで自動調整（flex:1） | Cards |
| `.IsFix` | 固定幅 --itemW指定 | Cards |

### FlexRのValueクラス

クラス名が比率を表す。2種類のコンテンツの比率を制御するラッパー。

| クラス | 左：右比率 | 説明 |
|--------|-----------|------|
| `.Flex55` | 50%:50% | 等分 |
| `.Flex46` | 40%:60% | 左4、右6 |
| `.Flex64` | 60%:40% | 左6、右4 |
| `.Flex37` | 30%:70% | 左3、右7 |
| `.Flex73` | 70%:30% | 左7、右3 |
| `.Flex28` | 20%:80% | 左2、右8 |
| `.Flex82` | 80%:20% | 左8、右2 |

25%など細かい調整は近いクラスを選び`--few`を上書きする。

## Tailwind CSSとの連携

### 役割分担

| 層 | Unit | Tailwind CSS |
|---|---|---|
| **レイアウト骨格** | ✅ | ❌ |
| **コンポーネント構造** | ✅ | ❌ |
| **色・グラデーション** | ❌ | ✅ |
| **余白微調整** | ✅(変数) | ✅ |
| **フォントサイズ** | ✅(変数) | ✅ |
| **レスポンシブ** | ✅ | ✅ |

**基本方針**: Unitクラスで骨格を作り、Tailwindで装飾を加える。

### 使用例

**Cards（3カラム）**:
```html
<div class="Cards col3">
  <div class="item gap-0 ">
    <figure><img src="/images/960x480.png" alt=""></figure>
    <div class="p-4 rounded-lg shadow-lg flex-1">
      <div>
        <h3 class="font-bold mb-2">カード 1</h3>
        <p>ここで Tailwind クラスを使って装飾できます。</p>
      </div>
    </div>
  </div>
  ...
</div>
```

**Toggle（アコーディオン）**:
```html
<details class="Toggle IsQa">
  <summary class="">これはどのような仕組みですか？</summary>
  <div class="">
    <p>summary と details タグを使った開閉コンテンツです。</p>
  </div>
</details>
```

Tailwindのarbitrary value構文を使用：

- `bg-[var(--MC)]` - メインカラーの背景
- `text-[var(--TC)]` - テキストカラー
- `border-[var(--SC)]` - サブカラーのボーダー

## 変数設計

### 基本レイアウト変数

| 変数 | 例 | 説明 | 使用シーン |
|------|----------|------|-----------|
| `--wid` | `1080px` | `main>*`のwidthになる幅 | コンテンツ幅制御 |
| `--gap` | `2rem` | Flexコンテンツの基本ギャップ | Cards, Panel, FlexR系 |
| `--MY` | `4rem` | セクション間の基本縦余白 | `main>*+*`, `.wrapper>*+*` |
| `--PX` | `1.5rem` | 全コンテンツの基本横余白 | `main`のpadding-inline |
| `--into` | 計算値 | 中身が--widの幅になる横余白 | `.intoL`, `.intoR` |
| `--out` | 計算値 | コンテンツ幅を画面幅にする | `.out`, `.outR`, `.outL` |
| `--imgW` | `30%` | 画像幅比率 | ImgText, Panel |
| `--itemW` | 計算値 | Cardsアイテム幅 | `.Cards`クラス |
| `--few` | 30% | FlexRの小さい要素幅 | Flex46, Flex37など |

### 色変数

| 変数 | 値（例） | 説明 |
|------|----------|------|
| `--mc` | `#2db542` | メインカラー |
| `--sc` | `#3194c9` | セカンダリーカラー |
| `--ac` | `#512db5` | アクセントカラー |
| `--bc` | `#fbf9ef` | ベースカラー |
| `--tc` | `#333` | テキストカラー |
| `--gr` | `#666` | グレー |
| `--bk` | `#000` | ブラック |
| `--wh` | `#fff` | ホワイト |
