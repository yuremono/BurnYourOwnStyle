# Style 設計書

本ドキュメントは、Burn Your Own Style システムの**設計思想・基本コンセプト・変数設計**を定義します。

各 Unit（Cards, Panel, ImgText など）の詳細な使用方法については [`unit.md`](./unit.md) を参照してください。

---

## 設計思想と基本コンセプト

### レスポンシブ設計方針

- **Unit クラス**: DesktopFirst レスポンシブ対応（`@max-md` 等を使用）
- **Tailwind CSS**: デフォルトの MobileFirst（`md:`, `sm:` 等）と併用可能

### コンテンツ幅の制御原理

**「基本コンテンツ幅を `--wid` で設定し、画面幅がそれ以上の時は中央配置」**

```scss
main>* {
    width: var(--wid);  // 固定幅（小さい画面でも--wid を維持）
    max-width: 100%;    // 画面より広くなったら縮小
    margin-inline: auto; // 中央配置
}
```

**なぜ `width: var(--wid)` と `max-width: 100%` なのか？**

1. **`width` の方が短い**
   - `width: 100%` と `max-width: var(--wid)`の挙動はほとんどの場合同じである

2. **`max-widthは多くの要素で基本値として使う`**
   - `width: var(--wid)` で design データに基づいた幅を設定
   - `max-width: 100%` で小さい画面でもコンテンツがはみ出さないよう制御

### 左右余白の計算原理

**`--into` と `--out` の関係**：

コンテンツを中央に配置し、`--wid` の幅を維持するための計算式は以下のように設計されている：

```scss
/* 左右のマージン（コンテンツ幅を画面幅にする） */
--out: calc(50% - 50vw);
/* 左右のパディング（中身が--wid の幅になる） */
--into: calc(50vw - clamp(0px, 50vw - var(--PX), (var(--wid) / 2)));
```

**`--into` の計算式の読み解き**：

この式は「画面幅が `--wid` より広い時にのみ、中央配置の余白を計算する」という仕組みです：

- `50vw - X` の形式で、左右から半分の余白を引く
- `clamp(0px, 50vw - var(--PX), (var(--wid) / 2))`：
  - 最小値：`0px`（画面が狭い時は余白なし）
  - 中間値：`50vw - var(--PX)`（実際の計算用）
  - 最大値：`var(--wid) / 2`（1080px/2 = 540px）

つまり、**画面幅が 1080px 以上の時は `--wid/2` の余白を計算し、それ以下の時は `0px` になる**という仕組みです。

**`--out` の計算式**：

```scss
--out: calc(50% - 50vw);
```

これは「コンテンツ幅が画面幅になるように」計算する式です。`50%` から `50vw` を引くことで、画面幅がコンテンツ幅よりも広くなった時のみ、その差分をマージンとして扱います。

**フォールバック値について**：

変数使用時は `var(--PX, 20px)` のようなフォールバック値は含めず、シンプルに `var(--PX)` と記載します。フォールバック値を指定すると：

- 変数が未設定の場合、フォールバック値が適用されるため「変数が定義されていない」という問題がわかりにくくなる
- 記述量が増え、可読性が低下する

そのため、**すべての変数は必ず `:root` で定義することを前提**とし、フォールバック値は省略します。

## 変数設計の基礎

### フォントサイズの変数設計

本システムでは、**基本フォントサイズは `rem` 単位で指定**し、タイトル（`h1`, `h2`, `h3` 等）のみ流動的な値（`clamp()`）を使用します。

```scss
:root {
    --bodyFZ: 1rem;
    @media (max-width: 834px) {
        --bodyFZ: 0.875rem;
    }
    --h1FZ: clamp(24px, 4.8vw, 48px);
    --h2FZ: clamp(20px, 3.2vw, 32px);
    --h3FZ: clamp(18px, 2.4vw, 24px);
}
```

**この設計の意図**：

1. **デザインデータに基づく変数定義**
   - 今後設定する「デザインから変数を抽出するスキル」により、デザインデータからすべてのフォントサイズを自動的に取得
   - `h2`, `h3` などのサイズもあらかじめ変数で定義済み

2. **Tailwind CSS との連携**
   - デザインから読み取ったサイズを変数で定義しておくため、**ほとんどのフォントサイズを Tailwind CSS で変更する必要がない**
   - Tailwind の `text-xl` 等のクラスを使用する場合でも、内部で変数を参照する形式にする

### 色変数の設計

**デザインで使われている色はほとんどを変数に置き換え**、あらかじめ `:root` で定義する：

```scss
:root {
    --mc: #2db542;  // メインカラー
    --sc: #3194c9;  // サブカラー
    --ac: #512db5;  // アクセントカラー
    --bc: #fbf9ef;  // ベースカラー
    --tc: #333;     // テキストカラー
    // ...
}
```

**Tailwind CSS との連携**：

Tailwind の arbitrary value 構文を使用し、CSS 変数を直接参照できます：

- **background-color**: `bg-[var(--mc)]`（メインカラー）
- **text-color**: `text-[var(--tc)]`（テキストカラー）
- **border-color**: `border-[var(--sc)]`（サブカラー）

この設計により、**デザインデータを一元的に管理**し、色やフォントサイズの変更も変数だけで完結できるようになります。

## 基本変数

### レイアウト変数

| 変数名 | デフォルト値 | 説明 | 使用シーン |
|--------|------------|------|-----------|
| `--gap` | `2rem` | FleX コンテンツの基本ギャップ | Cards, Panel, FlexR 系 |
| `--wid` | `1080px` | `main>*` の width になる幅 | `main>*` |
| `--MY` | `4rem` | セクション間の基本縦余白 | `main>*+*`, `.wrapper>*+*` の margin-top |
| `--PX` | `1.5rem` | 全てのコンテンツの基本横余白 | `main` の padding-inline |
| `--into` | 計算値 | 中身が--wid の幅になるように横余白を指定 | `.intoL`, `.intoR` |
| `--out` | 計算値 | コンテンツ幅を画面幅にする | `.out`, `.outR`, `.outL` |
| `--imgW` | `30%` | 画像幅比率 | ImgText, Panel で使用 |
| `--itemW` | 計算値 | Cards アイテム幅 | `.Cards` クラスで使用 |
| `--few` | 計算値 | FlexR の小さい要素幅 | Flex46, Flex37 など |

### 派生変数

以下の派生変数は、基本変数から計算される値で、実際に使用されています：

| 変数名 | 計算式 | 説明 |

| `--gapN` | `calc(var(--gap)*-1)` | ギャップの負の値（ネガティブマージン用） |
| `--gapH` | `calc(var(--gap)/2)` | ギャップの半分 |

## 基本的な構造とユーティリティ

### `main>*` と `.wrapper` の役割

**`main>*`（ページの主要コンテンツ）**：
- デフォルトで `--wid`（1080px）の幅を持つ
- 自動的に中央配置される
- 要素間の縦余白は `--MY` が適用される

**`.wrapper`（セクション用ラッパー）**：
- セクション全体を囲む時に使用
- 上下の padding を `--MY` で制御
- 左右の margin を `--out` で制御（コンテンツ幅を画面幅にする）
- 内部要素は自動的に中央配置される

**`.wid`, `.PX`, `.MY`（部分適用用ユーティリティ）**：
- `main>*` や `.wrapper` ではない要素に、部分的に変数を適用したい時に使用
- 例：`.wid` で強制的に 1080px 幅にする、`.PX` で左右パディングを指定するなど

**`.intoL`, `.intoR`, `.out`, `.outR`, `.outL`（左右個別制御用）**：
- `padding-inline` や `margin-inline` ではなく、左右別々に制御したい時に使用
- 例：`intoL` は左側のみのインナーパディング、`outR` は右側のみのアウトマージン
