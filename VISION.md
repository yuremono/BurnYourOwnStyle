# Burn Your Own Style VISION

## プロジェクトの目的

ユーザーが Web サイト制作で蓄積してきたスタイルシステム（クラス、変数、スタイリングの癖）を、Claude Code・Cursor 等のエージェントに理解させることで、Web 制作の全ての作業をエージェントに任せることを目標としたプロジェクトです。

完成後はこのプロジェクトを丸ごとコピーして新規プロジェクトの基盤として使用することを想定しています。`.claude/skills`, `/scss`, `/js` が主要ディレクトリになります。

## 基本コンセプト

### 1. 暗黙知の共有

**基本理念**：使い慣れた人間と、ルールドキュメントを読み込んだエージェント双方が暗黙知を持つことを前提とし、人間のタイピングコストを最小限に抑える。

- **クラス名・変数名の命名規則**：意味的な命名を最小限に、短縮・略語を積極的に使用
  - 主要な少数のクラス(スキル)を使い。その名前はユーザーが決めるのでエージェントが名称を決めることはない

- **学習コストの配分**：
  - エージェント：MD ファイルで網羅的に定義（クラス名、役割、使用方法）を把握し、skill実行時はコンポーネント毎の`skill`に準じてユーザーの指示を変数に置き換えて実装する
  - 人間：システム全体を理解する暗黙知
  - 境界：基本は MD で教えるが、慣習的な部分は人間が暗黙知として掌握

### 2. Tailwind.css との役割分担

**基本方針**：Burn Your Own Style = レイアウト固定、細部 = Tailwind

| 層 | Burn Your Own Style | Tailwind.css |
|---|---|---|
| **レイアウト骨格** | ✅ flex/grid 構成 | ❌ |
| **コンポーネント構造** | ✅ 構造定義 | ❌ |
| **色・グラデーション** | ❌ | ✅ |
| **余白微調整** | ✅(変数) | ✅ |
| **フォントサイズ** | ✅(変数) | ✅ |
| **レスポンシブ** | ✅ | ✅ |

## アーキテクチャ

### コンポーネント設計

**設計思想**：
- コンポーネント自体は「それ自体でセクションを成立させる」または「カードの集合体」程度の大きさ
- セクション単位：例／Hero セクション、Cards セクション、Accordion セクション
- Modifiers（モディファイア）はユーザーがその場面に応じて選択的に指定

### 命名規則

**ベースクラス**（初期段階）：
- `Cards` → `.Cards`
- `Accordion` → `.Accordion`
- `Panel` → `.Panel`
- `ImgText` → `.ImgText`
- `FlexRatio` → `.Flex55`, `.Flex46` ＊一部コンポーネントはスキル名=クラス名ではなくクラス名が変数、比率などを表す。

**モディファイア**（`__*` 形式）：
- `.__layer`：レイアウ トモード
- `.__qa`：Q&A スタイル
- `.__flow`：矢印ありの流れ
- `.__rev`：反転設定

**数値指定**（接頭辞 + 値）：
- `.col3`, `.col4`：カラム数
- `.Flex55`, `.Flex46`：左右比率
- `.img20`, `.img30`：画像比率

**ブレイクポイント制御**：
- `.bp-sm`：スマートフォン用挙動

## エージェントワークフロー

### ユーザー指示の構文
```
[対象] を [スキル名] で、[Modifier...] 、[--変数=値...] 、[出力形式] で
```

| 引数 | 必須 | 例 | 説明 |
|---|---|---|---|
| 対象 | ✅ | `hero.pen の MainVisual` / `ファイル名.pen のレイヤー名` | デザインファイルとレイヤー名。ファイルなしの場合は省略可 |
| スキル名 | ✅ | `Cards` `Accordion` `Panel` | `.claude/skills/` 配下のスキル名（パスカルケース） |
| Modifier | ⬜ | `col3` `__rev` `__layer` | スキルごとに定義。複数指定可 |
| 変数 | ⬜ | `--gap=60px` `--wid=80%` | 省略時はSKILL.mdのデフォルト値を使用 |
| 出力形式 | ⬜ | `React` `Astro` `HTML` | 省略時はプロジェクトのデフォルトに従う |

**例：**
```
hero.pen の MainVisual を Cards で、col3、--gap=60px、React で
```

---

### 基本フロー
```
1. 指示を受けたらSKILL.mdを参照する
   → .claude/skills/{スキル名}/SKILL.md
   → クラス構造・Modifier・変数・デフォルト値を把握する

2. デザインファイルが指定されていれば参照する
   → designs/{ファイル名}.pen
   → 指定レイヤーのデザインを視覚的に確認する
   → ファイルが存在しない場合はスキップする

3. 指示の引数をSKILL.mdの定義にマッピングする
   → 省略された引数はSKILL.mdのデフォルト値で補完する
   → 不明・矛盾がある場合は実装前にユーザーに確認する

4. コンポーネントを生成する
   → スキルで定義されたクラス構造を使用する
   → デザインの再現にはTailwindで装飾する
   → スキルのクラス・変数をTailwindで上書きしない
```

## 技術スタック

### 想定環境

- **HTML**：基本構造
- **React**：コンポーネントベース開発
- **Astro**：スタティックサイト生成
- **Tailwind.css**：細部スタイリング、装飾上書き
- **SCSS**：Burn Your Own Style 本体

### 設計思想

1. **Flexbox 中心のレイアウト**
   - 柔軟な配置を確保
   - カラム数で動的に幅を計算
   ＊ gridレイアウトは設計思想に合わないため基本レイアウトでは使用しない予定(__layerは例外)

2. **CSS 変数によるカスタマイズ**
   - `--gap`, `--wid`, `--PX` など
   - 状況に応じた微調整を提供

3. **レスポンシブ対応**
   - レイアウトコンポーネントではDesktop First (max-md)
   - TailwindはデフォルトのMobile First (md-,sm-)
   - モディファイアで挙動を細調整

4. **Tailwindで上書き可能**
   - レイアウトコンポーネントはTailwindのシングルクラスで上書きできるようcssファイル位置、または詳細度を調整
